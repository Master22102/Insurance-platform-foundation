## 1. Database — migrations applied today

### `tier_a_rpcs_and_policies`
- INSERT / UPDATE policies on `quick_scan_results` and `user_signal_profile_versions` (account-scoped via `auth.uid() = account_id`). Closed the write path RLS was previously blocking.
- RPCs (all `SECURITY DEFINER`, owner-guarded):
  - `propose_signal_profile(p_account_id, p_parsed_payload, p_source_voice_artifact_id)`
  - `confirm_signal_profile(p_version_id)`
  - `upsert_anchor_state(p_account_id, p_anchor_path)`
  - `record_terms_acceptance(p_account_id, p_version)`
  - `record_privacy_acceptance(p_account_id, p_version)`
  - `upsert_quick_scan_result(p_account_id, p_itinerary_hash, p_payload, p_trip_id)` — PRIMARY KEY `hash_key = '${account_id}:${itinerary_hash}'` for ON CONFLICT upsert.
  - `record_location_ping(...)` — certainty stamp (confirmed / likely / approximate).

### `tier_b_clause_timer_and_disruption_selector`
- `emit_elapsed_clause_timers() RETURNS integer` — scans `clause_timers` where `elapsed_at IS NULL AND due_at <= now()`, emits `clause_timer_elapsed` to the event ledger, stamps `elapsed_at`.
- `select_disruption_option(p_incident_id, p_option_id, p_actor_id) RETURNS uuid` — ownership via `incidents → trips → account_id`, sets `incidents.selected_option_id`, emits `disruption_option_selected`.
- `resolve_jurisdiction(p_incident_id) RETURNS jsonb` — returns jurisdiction snapshot from `incidents.jurisdiction_id` joined to `jurisdiction_references`.

### `clause_timer_scheduler` (doctrine catch-up)
- `CREATE EXTENSION IF NOT EXISTS pg_cron`.
- Unschedules any prior `clause_timer_tick` job then `cron.schedule('clause_timer_tick', '*/5 * * * *', 'SELECT public.emit_elapsed_clause_timers();')`.
- Safe to re-run; job id lookup via `cron.job` view.

### `mock_disruption_options_connector` + `mock_disruption_options_fix_event_log` (doctrine catch-up)
- `populate_mock_disruption_options(p_incident_id) RETURNS jsonb`, `SECURITY DEFINER`.
- Stamps `incidents.live_options_result` with a payload that carries `connector_source='mock'` and an explicit disclaimer string so UI cannot ambiguously render it as live.
- Flips `options_engine_activated=true`, sets `options_engine_trigger='mock_connector'`, writes `live_options_retrieved_at` and a 30-minute `live_options_expires_at`.
- Emits `live_options_mock_populated` to `event_logs` using the actual schema shape (`related_entity_type`, `related_entity_id`, `actor_id`, `actor_type='user'`, `event_data`).
- Documented replacement contract so a future real connector drops in without UI changes.

### Existing RPCs reused (no duplication)
`add_deep_scan_credits`, `initiate_deep_scan`, `create_claim_packet_from_incident`, `invoke_claim_packet`, `evaluate_trip_readiness`, `confirm_trip_readiness`, `evaluate_statutory_rights`, `change_incident_status`, `create_incident`, `register_evidence`.
