## 1. Database — migrations applied

### `tier_a_rpcs_and_policies`
- INSERT / UPDATE policies on `quick_scan_results` and `user_signal_profile_versions` (account-scoped via `auth.uid() = account_id`). Closes the write path that RLS was previously blocking.
- RPCs (all `SECURITY DEFINER`, owner-guarded):
  - `propose_signal_profile(p_account_id, p_parsed_payload, p_source_voice_artifact_id)` — writes a pending version row.
  - `confirm_signal_profile(p_version_id)` — flips pending → confirmed.
  - `upsert_anchor_state(p_account_id, p_anchor_path)` — idempotent path write.
  - `record_terms_acceptance(p_account_id, p_version)` / `record_privacy_acceptance(p_account_id, p_version)` — emit to `event_logs`.
  - `upsert_quick_scan_result(p_account_id, p_itinerary_hash, p_payload, p_trip_id)` — PRIMARY KEY `hash_key = '${account_id}:${itinerary_hash}'` for ON CONFLICT upsert.
  - `record_location_ping(...)` — location certainty stamp (confirmed / likely / approximate).

### `tier_b_clause_timer_and_disruption_selector`
- RPCs:
  - `emit_elapsed_clause_timers() RETURNS integer` — scans `clause_timers` where `elapsed_at IS NULL AND due_at <= now()`, emits `clause_timer_elapsed` to the event ledger, stamps `elapsed_at`.
  - `select_disruption_option(p_incident_id, p_option_id, p_actor_id) RETURNS uuid` — ownership via `incidents → trips → account_id`, updates `incidents.selected_option_id`, emits `disruption_option_selected`.
  - `resolve_jurisdiction(p_incident_id) RETURNS jsonb` — returns jurisdiction snapshot from `incidents.jurisdiction_id` joined to `jurisdiction_references`.

Existing RPCs reused (not duplicated):
`add_deep_scan_credits`, `initiate_deep_scan`, `create_claim_packet_from_incident`, `invoke_claim_packet`, `evaluate_trip_readiness`, `confirm_trip_readiness`, `evaluate_statutory_rights`, `change_incident_status`, `create_incident`, `register_evidence`.
