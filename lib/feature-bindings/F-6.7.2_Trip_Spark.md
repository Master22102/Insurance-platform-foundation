# Feature Binding — F-6.7.2 Trip Spark

**Status:** Draft implementation prompt. Not yet built.
**Depends on:** F-6.7.1 Field Notes (schema must land first), Trip Draft
Engine (F-6.5.17), Deep Scan (F-6.5.13), Checklist Screen Family
(S-CHECKLIST-001).
**Does not depend on:** New trip infrastructure. Trip Spark calls the
existing `create_trip_draft` pathway — no parallel trip model.

---

## 1. Purpose

When a user's Field Notes cluster around a region, Trip Spark detects the
concentration and offers to convert the cluster into a trip draft with
pre-populated destinations and linked note IDs. It is **not** a separate
trip type; it is a fast path into the existing Trip Draft Engine.

## 2. Data classification

- Cluster records: **T2** (derived personal data). Account-scoped.
- Cluster centroid coordinates: **T2** — never exposed via unauthenticated
  APIs.

## 4. RLS (four policies, guardrail §2)

- SELECT: account membership.
- INSERT: system-only (SECURITY DEFINER function inserts; no direct policy
  for `authenticated`).
- UPDATE: SECURITY DEFINER only for status transitions. A separate policy
  allows the user to mark a cluster dismissed via RPC (still SECURITY
  DEFINER).
- DELETE: none. Clusters age out to `stale` status; never deleted.

## 5. Clustering engine

### Trigger points
1. After `create_field_note` commits, enqueue `recompute_spark_clusters(account_id, region_code)` on the background runner (Supabase cron or edge function on a short debounce).
2. On demand via `/account/field-notes` "Refresh clusters" CTA.

### Algorithm (Phase 1 — deterministic, no ML)
- Group all non-archived `field_notes` for the account by `region_code`.
- For each region with ≥3 notes across ≥2 distinct categories, compute
  `cluster_score = notes_count * 0.6 + distinct_categories * 1.2 + recency_boost`.
- Upsert a `trip_spark_clusters` row per qualifying region.
- Threshold for surfacing as a suggestion: `cluster_score >= 5`.

### Outputs
- A `trip_spark_cluster_formed` event per new cluster.
- A `trip_spark_suggestion_fired` event the first time a cluster crosses
  the surfacing threshold.

## 6. RPCs

- `recompute_spark_clusters(p_account_id, p_region_code)` — idempotent
  reclustering; upserts clusters, updates `note_count` /
  `category_histogram` / `cluster_score`.
- `convert_spark_cluster_to_trip(p_cluster_id)` — calls the existing
  `create_trip_draft` RPC with `destination_summary` derived from the
  cluster, then updates the cluster's `status = 'converted'` and
  `converted_trip_id`. Also calls `attach_field_note_to_trip` for each
  member note.
- `dismiss_spark_cluster(p_cluster_id, p_reason)` — sets `status =
  'dismissed'`, `dismissed_at = now()`.

## 7. Event types (feature_id F-6.7.2)

- `trip_spark_cluster_formed` (info)

## 10. Cross-feature dependency declarations

Writes: `trip_spark_clusters`, `field_notes.cluster_id`, `event_ledger`.
Reads: `field_notes`, `accounts`, `account_members`, `trips`.
Calls (existing RPCs): `create_trip_draft`, `attach_field_note_to_trip`,
`generate_trip_checklist`.
Downstream: Deep Scan Axis 5 lightweight query on cluster crossing; Trip
Draft Engine readiness gate.

## 11. Acceptance criteria

- [ ] Clustering is deterministic: same inputs yield identical cluster
      records on re-run.
- [ ] Threshold crossing fires `trip_spark_suggestion_fired` exactly once
      per cluster until dismissed or converted.
- [ ] Convert action creates a trip draft that appears in the existing
      trip list with correct destinations and linked notes.
- [ ] Dismiss action does not re-surface the same cluster for 14 days even
      if the score climbs (cooldown enforced via `dismissed_at`).
- [ ] Replay coverage: cluster lifecycle reconstructs from events.

## 12. Out of scope

- ML-based clustering across categories or across regions (Phase 2).
- Multi-account / shared cluster computation (never — account-scoped
  always).
- Automatic conversion without user consent (explicit CTA always).
