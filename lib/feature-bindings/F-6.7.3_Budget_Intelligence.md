# Feature Binding — F-6.7.3 Budget Intelligence

**Status:** Draft implementation prompt. Not yet built.
**Depends on:** `user_profiles.preferences` JSONB (exists), voice parse
pipeline (exists, 7 contexts), Per-Trip Unlock Model (F-1, exists), Trip
Spark (F-6.7.2, upcoming).
**Key architectural note:** Lives **inside** `user_profiles.preferences`
JSONB as a structured sub-object. **Does not** get its own table. Voice
artifacts already write here; Budget Intelligence extends the same path.

---

## 1. Purpose

Category-level spend model that lets the platform say "a 10-day Italy trip
at your comfort level costs roughly $X" — turning stated preferences into
trip unlock conversion moments and coverage-gap conversations.

## 2. Data classification

- Budget bands / category preferences: **T2** (personal operational).
  RLS already gates `user_profiles` by `auth.uid()`.
- Actual historical spend (if ever captured): **T1** (financial).
  **Out of scope for MVP** — no historical spend tracking until an ADR
  explicitly approves T1 handling.

## 3. Schema — JSONB contract under `user_profiles.preferences.budget`

```json
{
  "budget": {
    "version": 1,
    "updated_at": "2026-04-21T00:00:00Z",
    "source": "voice_capture" | "manual_entry" | "inferred",
    "currency": "USD",
    "per_night_lodging": { "min": 120, "typical": 180, "max": 260 },
    "per_day_food": { "min": 30, "typical": 60, "max": 120 },
    "per_day_activities": { "min": 0, "typical": 50, "max": 150 },
    "per_day_local_transit": { "min": 0, "typical": 20, "max": 60 },
    "total_trip_cap": { "currency": "USD", "amount": 4000 },
    "comfort_level": "budget" | "moderate" | "comfort" | "premium",
    "flex_flags": ["splurge_on_food", "budget_on_lodging"]
  }
}
```

A JSON schema lives at `lib/schemas/budget-preferences.schema.json` and is
validated by the RPC below before writing.

### Migration (`<ts>_f_6_7_3_budget_intelligence.sql`)
No new table. Adds:
- A `CHECK` constraint that if `preferences -> 'budget' -> 'version'` is
  present, it is a positive integer.
- An expression index: `((preferences -> 'budget' ->> 'comfort_level'))`
  to support analytics aggregations.

## 4. RPCs

- `set_budget_preferences(p_budget jsonb)` — validates against the JSON
  schema, sets `updated_at` server-side, writes to
  `user_profiles.preferences.budget`. Emits event.
- `estimate_trip_cost(p_trip_id)` — reads destination cost table (see §5)
  and the user's `budget.typical` values; returns a range object.
- `get_budget_preferences()` — returns the current `budget` sub-object or
  a default stub for a new user.

All SECURITY DEFINER, search_path = ''.

## 5. Reference data — `destination_cost_cache` (new table)

## 6. Event types (feature_id F-6.7.3)

- `budget_preferences_set` (info)
- `budget_preferences_updated_from_voice` (info)
- `budget_preferences_inferred` (info) — reserved; not used until inference
  logic lands
- `trip_cost_estimated` (info)

## 7. UI surfaces

- `/account/preferences/budget` — sliders per category with comfort-level
  chips (budget/moderate/comfort/premium) that auto-set band defaults.
- Voice capture (context `preference_capture`) — recognized utterances
  like "I usually don't spend more than $180 a night" write to this
  structure via `set_budget_preferences`.
- Trip Spark suggestion banner — "Likely $X–$Y based on your budget" using
  `estimate_trip_cost`.
- Trip Draft page — budget chip next to destination; tap to adjust.

## 9. Cross-feature dependencies

Writes: `user_profiles.preferences.budget`, `destination_cost_cache` (via
migration), `event_ledger`.
Reads: `user_profiles`, `trips`, `route_segments`,
`destination_cost_cache`.
Consumed by: Trip Spark (cost chip), Trip Draft page (per-leg estimate),
Per-Trip Unlock CTA ("Unlock and start building for $Y").

## 10. Acceptance criteria

- [ ] `set_budget_preferences` rejects any payload missing required keys.
- [ ] Voice parse in `preference_capture` context writes deterministically
      and is idempotent on replay.
- [ ] `estimate_trip_cost` returns a range in ≤50ms for trips with ≤10
      destinations.
- [ ] `/account/preferences/budget` renders existing values on re-entry
      (no silent reset).
- [ ] All four event types appear in `event_type_registry` with
      feature_id `F-6.7.3`.

## 11. Out of scope

- Actual historical spend import (T1; blocked on ADR).
- Live pricing from OTA APIs (Stay Range Engine F-6.7.5 territory).
- Multi-currency conversion UX (MVP: store USD-equivalent index; UI
  displays user's preferred currency).
