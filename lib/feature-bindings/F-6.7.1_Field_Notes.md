## 1. Purpose

Field Notes is the Saved Intelligence Layer — the persistent, structured
capture surface where a user saves any planning-relevant object (creator
video, Deep Scan signal, pasted URL, voice clip, manual note) with enough
metadata that downstream engines (Trip Spark, Activity Auto-Population,
Checklist, Coverage Graph) can act on it.

Field Notes is **not** a note-taking app. It is the raw input layer for the
Coverage Intelligence Loop.

## 2. Data classification

- Note body: **T2** (personal operational data). Account-scoped RLS.
- Imported URL metadata: **T3** (platform metadata extracted from public
  sources). Still account-scoped for ownership but not secret.
- Any T1 content (passport number, health doc) pasted into a note MUST be
  rejected by a pre-insert guard; notes are not a safe surface for T1.

## 4. RLS policies (per guardrails §2)

Four policies, no `FOR ALL`, no `USING (true)`:

```sql
CREATE POLICY "Members can select own account's field notes"
  ON field_notes FOR SELECT TO authenticated
  USING (account_id IN (SELECT account_id FROM account_members WHERE user_id = auth.uid()));

CREATE POLICY "Members can insert field notes into own account"
  ON field_notes FOR INSERT TO authenticated
  WITH CHECK (created_by = auth.uid()
    AND account_id IN (SELECT account_id FROM account_members WHERE user_id = auth.uid()));

CREATE POLICY "Creator can update own field notes"
  ON field_notes FOR UPDATE TO authenticated
  USING (created_by = auth.uid())
  WITH CHECK (created_by = auth.uid());

CREATE POLICY "Creator can soft-archive own field notes"
  ON field_notes FOR UPDATE TO authenticated
  USING (created_by = auth.uid())
  WITH CHECK (created_by = auth.uid() AND archived_at IS NOT NULL);
```

No DELETE policy. Hard delete is forbidden; soft-archive only.

## 5. RPCs (SECURITY DEFINER, search_path = '')

- `create_field_note(p_source_type, p_source_reference_id, p_title, p_body, p_region_code, p_categories, p_seasonality, p_source_url)` → returns `note_id`. Validates caller, rejects any content matching a T1 pattern (SSN, passport regex, card number).
- `attach_field_note_to_trip(p_note_id, p_trip_id)` → validates membership on both.
- `archive_field_note(p_note_id)` sets `archived_at = now()`.
- `import_url_as_field_note(p_url)` → queues a URL extraction edge function (see §7).

## 7. Edge function `extract-url-metadata`

- Input: `{ url, account_id, note_id }`
- Fetches the URL server-side, extracts `og:title`, `og:description`,
  `og:image`, `og:site_name`, `og:locale`, any jsonld Place or Event
  objects, and a sanitized description.
- Writes the extracted metadata back via
  `update_field_note_extracted_metadata` RPC.
- CORS per guardrails §8.
- Never fetches more than 2 MB; rejects non-HTML content-types.

## 10. Cross-feature dependency declarations (§9a per guardrails)

Writes to: `field_notes`, `field_note_attachments`, `event_ledger`,
`event_type_registry`, `feature_registry`.

Reads from: `accounts`, `account_members`, `trips`, `deep_scan_signals`,
`voice_artifacts`, `creator_discovery_results`.

Consumed by (downstream):
- **Trip Spark (F-6.7.2)** — clustering reads `field_notes.region_code` and
  `field_notes.categories`.
- **Activity Auto-Population (F-6.5.19)** — activity suggestion query joins
  `field_notes` for the current destination.
- **Checklist Screen Family** — destination-level checklist generation
  considers tagged field notes (e.g. cooking class → kitchen-safety item).
- **Coverage Graph (F-6.5.2)** — informational only for MVP; coverage
  implications flagged for Phase 2.

## 11. Acceptance criteria

- [ ] Migration applied; RLS enforced; unauthenticated select returns 0.
- [ ] `create_field_note` with an SSN-like regex in the body emits
      `field_note_t1_content_rejected` and returns an error.
- [ ] URL import populates title/region/category for at least 80% of the
      first 20 curated test URLs.
- [ ] `/account/field-notes` renders in ≤300ms for 500 notes.
- [ ] Trip Spark can query `field_notes` by `(account_id, region_code)` and
      cluster in one SQL call with a functional index.
- [ ] Replay coverage: every state change on `field_notes` emits a ledger
      event; replaying events reconstructs the table.

## 12. Out of scope (explicitly deferred)

- Full-text search (add GIN on tsvector in a Phase 2 migration).
- Shared-account collaboration on notes (notes are per-creator).
- AI summarization of imported URLs (possible Phase 2, needs model routing).
- Hard delete / GDPR erasure integration — follows M-005 once
  `erasure_requests` lands.
